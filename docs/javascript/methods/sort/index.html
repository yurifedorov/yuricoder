Метод sort()

Сортирует массив, меняя в нём порядок элементов.

          
          let arr = [ 'a', 'с', 'b' ];
alert(arr.sort());  // a, b, c

Было [ a, с, b ] стало [ a, b, c ] - метод sort() отсортировал элементы в нужном порядке.

По умолчанию элементы сортируются как строки в соответствии с лексикографическим порядоком (то есть сортирувка по-алфавиту)

Сортировка чисел

          
          let arr = [ 1, 2, 14 ];
alert(arr.sort());  // 1, 14, 2
Было [ 1, 2, 14 ] стало [ 1, 14, 2 ] - элементы отсортировались в неправильном порядке.

Все потому что при сравнении элементы-числа преобразовались в строки и согласно лексикографическому порядку 2 > 14

Собственный порядок сортировки

Нужно предоставить функцию в качестве аргумента arr.sort(function):

          
          function compareNumeric(a, b) {
  if (a > b) return 1; // если первое значение больше второго
  if (a == b) return 0; // если равны
  if (a < b) return -1; // если первое значение меньше второго
}

let arr = [ 1, 2, 15 ];
arr.sort(compareNumeric);
alert(arr);  // 1, 2, 15

Array Sort Comparison

          
          const arr1 = ['a', 'b', 'c'];
const arr2 = ['b', 'c', 'a'];

console.log(
  arr1.sort() === arr1,        // true
  arr2.sort() ==  arr2,        // true
  arr1.sort() === arr2.sort()  // false
);

Объяснение

Во-первых, метод sort() сортирует исходный массив, а также возвращает ссылку на этот массив. Это означает, что когда мы пишем arr2.sort(), объект массива arr2 сортируется.

Поскольку arr1.sort() и arr1 указывают на один и тот же объект в памяти, первый тест на равенство возвращает true.
Это верно и для второго сравнения: arr2.sort() и arr2 указывают на один и тот же объект в памяти.

В третьем тесте порядок сортировки arr1.sort() и arr2.sort() одинаков; однако они все еще указывают на разные объекты в памяти.
Таким образом, третий тест оценивается как false.
