Основные сетевые термины и сетевые модели

Что такое сеть?

Сеть - совокупность устройств и систем, которые соеденены друг с другом и общаются между собой.

Компоненты сети
оконечные узлы промежуточные устройства сетевые среды

устройства, которые передают и/или принимают какие-либо данные

  • компьютеры
  • телефоны
  • сервера
  • телевизоры
  • и т.д.

устройства, которые соединяют оконечные узлы между собой

  • коммутаторы
  • концентраторы
  • модемы
  • маршрутизаторы (роутеры)
  • точки доступа Wi-Fi

среды, в которых происходит непосредственная передача данных

  • кабели медные (посредством электрических сигналов)
  • кабели оптоволоконные (посредством световых имульсов)
  • беспроводные устройства (посредством радиоволн)
  • сетевые карты
  • коннекторы
  • воздушная среда передачи

Сетевые модели

  • модель OSI (эталонная модель, нужно знать как теорию)
  • модель TCP/IP (используется на практике)

Layer Тип данных Функции Примеры

Процесс взаимодействий

Пример клиента и сервера, который находится в одной локальной сети.

Сервер

  1. Application Layer

    • пользователь набирает адрес сайта
    • используется протокол HTTP
    • упаковка и переход на следующий уровень

  2. Presentation Layer

    • данные структурируются
    • данные приводятся в формат, который может быть прочитан на сервере
    • упаковка и переход на следующий уровень

  3. Session Layer

    • создается сессия между компьютером и сервером

  4. Transport Layer

    • установление соединения и контроль за принятыми данными
    • используется протокол TCP (требуется надежное соединение, т.к. это веб-сервер)
    • указываем порт, на который будем стучаться и порт источника (чтобы сервер знал, куда отправлять ответ)
    • стандартно — это 80 порт, а не на почтовый сервер
    • упаковка и переход на следующий уровень

  5. Network Layer

    • указывается, на какой адрес отправлять пакет
    • адрес назначения: пусть адрес сервера будет 192.168.1.2
    • адрес источника: пусть адрес компьютера 192.168.1.1
    • упаковка и переход на следующий уровень

  6. Data Link Layer

    • добавляются физические адреса источника и назначения
    • MAC-адрес компьютера
    • MAC-адрес сервера

  7. Physical Layer

    • полученные биты конвертируются в электрические сигналы и отправляются на сервер (если это витая пара)
    • далее какое-то время сигнал идет по проводам

  1. Application Layer

    • приложения или сервисы понимают, что надо выполнить

  2. Presentation Layer

    • видит, как все должно быть структурировано и приводит информацию в читабельный вид

  3. Session Layer

    • происходит установление сеанса между компьютером и сервером

  4. Transport Layer

    • проверяется порт назначения (не адрес)
    • по номеру порта, выясняется какому приложению или сервису адресованы данные
    • если это веб-сервер, то номер порта 80

  5. Network Layer

    • проверяется IP-адрес назначения
    • если он верен, данные поднимаются выше

  6. Data Link Layer

    • проверяется MAC-адрес назначения (ему ли это адресовано)
    • если да, то проверяется кадр на целостность и отсутствие ошибок
    • если все прекрасно и данные целы, он передает их вышестоящему уровню

  7. Physical Layer

    • электрический сигнал подошел к серверному железу
    • сингнал конвертируются в понятную битовую последовательность для канального уровня
