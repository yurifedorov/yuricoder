9. Определения методов (Method Definitions)

9.1. Безопасные и Idempotent методы

9.1.1. Безопасные методы

Программе следует информировать пользователя о любых действиях, которые могут иметь непредсказуемое значение для него или других лиц

Было принято соглашение, что методы GET и HEAD никогда не должны иметь иного значения, кроме загрузки. Эти методы следует рассматривать как "безопасные"

пункт 9.1.1. спецификации RFC 2068

9.1.2. Idempotent методы

Методы GET, HEAD, PUT и DELETE обладают свойством "idempotence"

P.S. пока я так и не понял что это за свойство.....

9.2. OPTIONS

Метод OPTIONS предназначен для получения поддерживаемых сервером возможностей

9.3. GET

Метод GET предназначен для получения существующего ресурса. В URL-адресе содержится вся необходимая информация для определения местонахождения и возвращения ресурса сервером.

9.4. HEAD

Метод HEAD подобен GET, однако не передается тело сообщения (только заголовки)

9.5. POST

Метод POST предназначен для создания нового ресурса. Запросы по методу POST обычно содержат данные для создания нового ресурса.

9.6. PUT

Метод PUT предназначен для обновления существующего ресурса. В содержимом могут находиться обновленные данные для ресурса.

9.7. DELETE

Метод DELETE предназначен для удаления существующего ресурса.

9.8. TRACE

Метод TRACE используется для получения от сервера информации о «прыжках»
